# Security policy

Gemini Security Skills includes dual-use cyber security guidance. This policy
defines the intended security posture for the repository.

## Supported use

The skills support:

- Authorized security assessments.
- Defensive operations.
- Incident response.
- Malware triage in isolated labs.
- Secure software engineering.
- DevSecOps hardening.
- CTF and lab-based exploit research.
- Threat modeling and risk reduction.

## Prohibited use

The skills must not be used to support:

- Unauthorized access.
- Credential theft.
- Persistence.
- Stealth.
- Evasion.
- Destructive actions.
- Malware deployment.
- Malware improvement.
- Real-world exploit weaponization.
- Automated exploitation of public targets.

## Reporting a security issue

If you find a dangerous instruction, unsafe workflow, or policy gap in this
repository, open an issue at:

```text
https://github.com/Masriyan/gemini-security-skills
```

Include:

- The affected file.
- The risky wording.
- Why it is unsafe.
- A safer replacement if you have one.

Do not include live target details, stolen data, malware payloads, credentials,
or exploit material that enables misuse.

## Maintainer review standard

Security-related changes should be reviewed for:

- Authorization language.
- Scope confirmation.
- Defensive outcome.
- Evidence handling.
- Avoidance of stealth, persistence, evasion, credential theft, and destructive
  behavior.
- Remediation and detection guidance.
